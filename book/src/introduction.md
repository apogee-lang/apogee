# The Apogee Book

Welcome. You're about to learn a programming language that was designed for the way software is actually built in 2026 — with AI writing most of the code, with concurrency as the default, and with bugs that should have been caught at compile time costing real companies real money every single day.

Apogee is a statically-typed language that compiles to Python. It has null safety, constraint types, structured concurrency, and `@intent` annotations. If that sentence made your eyes glaze over, don't worry — by the end of this book, you'll understand not just what those words mean, but *why they matter* and *how they change the way you write software*.

This book assumes you know how to program. You've written code in Python, JavaScript, Java, Rust, Go, or something similar. You know what a function is. You know what a type is. You've hit a `NullPointerException` or a `TypeError: cannot read property of undefined` at 2am and wanted to throw your laptop out a window.

Apogee exists so that never happens again.

## How to Read This Book

Chapters 1-4 are the foundation. Read them in order. After that, you can jump to whatever interests you — concurrency (Chapter 6), data processing (Chapter 7), or building real applications (Chapter 8).

Every code example in this book compiles and runs. Try them at [apogee-lang.dev](https://apogee-lang.dev) or install Apogee locally with `pip install apogee-lang`.

Let's begin.
